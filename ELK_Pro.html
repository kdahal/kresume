ELK Onboarding application - filebeat

Generally, frontend application uses beats to send logs directly to Logstash for further process and enhancing the data, before visualizing it in Kibana. Thre is a spicific API URL to be connected to frontend application to Logstash for transferring or parsing the logs data. Logstash alwars listens through Beats to incomming logs data through API and processes for further analyses.

Step 1. Certation of the filter Create a filter using Grok parser or Ruby script implement in the Logstash server which will help process the events.

Sample filter:

Onboarding application

input {
beats {
#host => logstash1.
port => 5xxx
# Enter 5 series port
ssl => true
ssl_key => '/app/logstash/cert-file/logstash.pkcs8.key'
ssl_certificate => '/app/logstash/cert-file/wildcard'
}
}
filter{
if([fields][log_type] == "xxxx")
# Tags of fields and log_type should match the configuration of filebeat
{
# write the filter
}
output{
if([fields][log_type] == "xxxx")
# tags of fields and log_type should match the configuration of filebeat
{
elasticsearch {
hosts => ["https://elasticsearch1:9200", "https://elasticsearch2:9200"]
cacert => '/app/logstash/cert-file/wildcard-ca.crt'
user => 'logstash_writer'
password => password
index =>"xxxx*"
}
stdout{}
}
}

Step 2. Login to the Logstash server as Logstash user $ su – logstash
Step 3. move to directory /app/logstash/config $ cd /app/logstash/config
Step 4. Create a new file (name after your application (we can use any name)) but it should end with extension .conf example: $ vim cs-iislogs.conf
Step 5. Copy the filter you have created and paste in the filter (We are using a port )
Step 6. Save and exit $:wq!
Step 7. Inorder to open the port in the server we need to restart the Logstash services $ sudo systemctl restart logstash
Step 8. To observe the log run the following command $ tail -f /var/log/messages
